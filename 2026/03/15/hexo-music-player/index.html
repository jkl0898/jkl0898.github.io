为 Hexo 博客添加跨页面音乐播放器

前言

最近想给博客加个音乐播放器，让访客在浏览文章时能听着音乐。找了一圈方案，最后选择了 APlayer + jquery-pjax 的组合，实现了全站跨页面音乐不中断的效果。

本文记录完整的实现过程。

效果预览

访问博客后，页面底部会出现一个固定的音乐播放器。点击播放后，无论你如何切换页面，音乐都会继续播放，不会被中断。

技术选型

技术 作用
APlayer 音乐播放器组件
jquery-pjax 无刷新页面切换，保持播放状态
jsDelivr CDN 加速音乐文件

实现步骤

1. 引入 APlayer

在页面中引入 APlayer 的 CSS 和 JS：

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/aplayer/dist/APlayer.min.css">
<script src="https://cdn.jsdelivr.net/npm/aplayer/dist/APlayer.min.js"></script>

2. 添加播放器容器

在 </body> 之前添加播放器容器：

<!-- 音乐播放器 -->
<div id="aplayer-fixed"></div>

3. 初始化播放器

创建 music-player.js：

(function($){
  function initAPlayer() {
    if (window.aplayer) return;
    
    const playerOptions = {
      container: document.getElementById('aplayer-fixed'),
      autoplay: false,
      loop: 'all',
      volume: 0.7,
      listFolded: true,
      audio: [
        {
          name: 'Canon in D',
          artist: 'Pachelbel',
          url: 'https://cdn.jsdelivr.net/gh/xxx/music/canon_in_d.mp3',
          cover: 'https://picsum.photos/300/300?random=1',
          lrc: ''
        }
      ]
    };
    
    window.aplayer = new APlayer(playerOptions);
  }
  
  $(document).ready(function(){
    initAPlayer();
  });
})(jQuery);

4. 引入 pjax 实现跨页面播放

关键点：使用 jquery-pjax 而不是原生 pjax

<script src="https://cdn.jsdelivr.net/npm/jquery-pjax@2.0.1/jquery.pjax.min.js"></script>

初始化 pjax：

(function($){
  if (typeof $.pjax === 'undefined') return;
  
  $(document).pjax('a[href^="/"]', {
    container: '#container',
    fragment: '#container',
    timeout: 8000
  });
  
  $(document).on('pjax:complete', function() {
    console.log('Pjax 切换完成，播放器继续播放');
  });
})(jQuery);

5. 音乐资源托管

音乐文件托管方案对比：

方案 优点 缺点
阿里云 OSS 稳定 需要备案
GitHub + jsDelivr 免费、无需备案 单文件 < 50MB

最终选择 GitHub + jsDelivr 方案：

  1. 将 MP3 文件放入项目的 music/ 目录
  2. 推送到 GitHub
  3. 使用 jsDelivr CDN 链接： https://cdn.jsdelivr.net/gh/用户名/仓库名@分支名/music/文件名.mp3

踩坑记录

问题1：pjax 库引用错误

症状： 控制台提示 "Pjax 库未加载"

原因： 使用了原生 pjax 库，但代码中用的是 $.pjax（jQuery 版本）

解决： 改用 jquery-pjax 库

问题2：OSS 链接强制下载

症状： 点击播放后直接下载文件，无法在线播放

原因： OSS 默认域名会触发强制下载

解决： 使用 GitHub + jsDelivr 替代

问题3：jsDelivr 404 错误

症状： jsDelivr 链接返回 404

原因： 分支名写错（写成 main，实际是 master）

解决： 检查并使用正确的分支名

总结

通过 APlayer + jquery-pjax + jsDelivr 的组合，实现了：

  • ✅ 全站底部固定播放器
  • ✅ 跨页面音乐不中断
  • ✅ 无需备案的音乐资源托管
  • ✅ CDN 加速，访问快速
